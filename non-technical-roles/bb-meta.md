## Abstract

Sassquatch discusses non-technical roles in RaidGuild and web3 work, focusing on how contributors can create value beyond writing code. The session highlights operations, project coordination, client communication, community work, and the practical paths for non-developers to participate in technical ecosystems.

## Transcript Summary

Sassquatch explains how non-technical contributors can create real value in RaidGuild and similar web3 communities. The session pushes against the idea that participation is only for developers, highlighting coordination, client communication, operations, cleric work, documentation, community support, and project ownership.

The practical message is that non-technical roles still require skill, initiative, reliability, and context. Contributors who understand people, process, and communication can help technical teams ship better work and can build meaningful paths into the guild without pretending to be engineers.
