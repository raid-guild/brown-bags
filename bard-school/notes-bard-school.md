From those plus Bard School, here’s a RaidGuild-specific pattern language draft:

1. `Proof Over Posture`
- publish evidence of work, not brand theater
- default artifact: teardown, retro, demo, build log, decision trace

2. `Seeds, Not Streams`
- every content cycle starts from one substantial source artifact
- everything else is derived from that seed

3. `Funnel By Intimacy`
- match message depth to relationship depth
- awareness gets signal, consideration gets proof, conversion gets specificity

4. `Institutional Memory As Infrastructure`
- treat notes, patterns, proposals, retros, and docs as operational systems
- if it matters repeatedly, encode it

5. `Pattern Before Automation`
- before building an agent or workflow, define the evaluation pattern first
- what counts as good, bad, complete, risky, missing

6. `Human Judgment At The Edge`
- automate screening and structuring
- keep final taste, ethics, and strategic calls with humans

7. `Specificity Preserves Identity`
- generalized tooling drifts toward sameness
- RaidGuild should encode its own language, standards, rituals, and heuristics explicitly

8. `Public Goods, Private Advantage`
- whenever possible, publish reusable frameworks
- the moat is not secrecy, it is taste, execution, and accumulated pattern depth

9. `Minimal Cadence, Maximum Signal`
- do the smallest sustainable content/program cadence that compounds
- consistency beats aspiration-heavy collapse
