## Abstract

This brown bag demos Droid as a coding-agent workflow for auditing documentation, planning fixes, and updating a repository from the terminal. The discussion focuses on practical developer ergonomics: readable agent responses, task breakdowns, repo inspection, documentation maintenance, and the overlap between Droid and other AI-assisted coding tools.

## Transcript Summary

The Droid session is a hands-on discussion of terminal-native coding agents. The speaker describes giving Droid a repo-wide documentation audit task, watching it inspect the codebase, produce a readable plan, and then apply fixes with little handholding. A major theme is the quality of the interaction: scannable responses, good task decomposition, and a workflow that works from the terminal rather than an IDE.

The group compares Droid with Cursor, Warp, OpenCode, and subscription-based model access. The practical value is not just that the tool can write code, but that it can reason through repo maintenance work, keep the developer oriented, and use models from Anthropic, OpenAI, or OpenRouter in a flexible setup.
